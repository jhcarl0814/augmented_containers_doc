struct augmented_sequence_t;

augmented_sequence_t is an augmented, general-purpose sequence. It's part of the augmented containers library, providing a more powerful version of sequence containers (let containers (and possibly its subranges) always have several accompanying results of algorithms/views), as well as <algorithm> and <ranges> (when the input sequence changes, refresh output values/ranges in logarithmic time complexity). To help understand what kind of problems the library solves: Dynamic problem (algorithms) - Wikipedia, Augmented map - Wikipedia.

  • augmented_sequence_t allows subrange update: first modify some elements, then call update_range to refresh the accumulated results that cover those elements in O(distance(range)+lg(N)) time. A common mistake would be to forget to call update_range.
  • augmented_sequence_t allows subrange query: call read_range to collect accumulated results that cover the range in O(lg(distance(range))) time. If the accumulated result is a range instead of a single value, the result is probably formed by accumulated results linking to each other, in this case you should not call read_range because there are no more pointers left for you to form another range (see examples below for an intuitive feel).
  • augmented_sequence_t allows inserting and erasing single element in O(lg(N)) time. Inserting a range of elements is implemented by 1 bulk loading operation (O(distance(range)) time) and 1 split operation followed by 2 concatenate operations. Erasing a range of elements is implemented by 2 split operations.
  • augmented_sequence_t allows spliting one sequence into two in O(lg(N)) time, and allows concatenating two sequences into one in O(lg(max(Nlhs, Nrhs))) time.
  • ⚠Insert, erase, split, concatenate operations automatically update accumulated results of the affected nodes. update_range is needed only after modification of elements through dereferencing iterators.

Examples

The untagged pointers are drawn as solid lines. The 0b1-tagged pointers are drawn as dashed lines.

In rb3p's examples, end node's child_right, parent and child_left are drawn separately to not break graphviz's layout.

In rb2p's examples, non-end node's loop (node's my_list_begin() → node's child_right, node's child_right's next() → node's leftmost_descendent_of_child_right, node's leftmost_descendent_of_child_right's my_list_begin() → node's rightmost_descendent_of_child_left, node's rightmost_descendent_of_child_left's my_list_begin() → node's child_left, node's child_left's next() → node, the last edge of the loop is 0b1-tagged) has it's edges drawn end-to-end to make it clearer (the pointers are still pointing to whole nodes, not members of nodes).

In rb2p's examples, end node's loop (end node's my_list_begin() → leftmost_descendent_of_root, leftmost_descendent_of_root's my_list_begin() → root, root's next() → rightmost_descendent_of_root, rightmost_descendent_of_root's my_list_begin() → end_node, the last edge of the loop is 0b1-tagged) is not drawn to not break graphviz's layout.

In rb2p's examples, end node has my_list_begin_, non-end node has my_list_begin_ and next_. The bit210 of my_list_begin_ stores the role of the node (there are 8 roles: end△, root▽, child_left_not_a_leftmost_descendent◤, child_left_leftmost_descendent_of_non_root◢, child_left_leftmost_descendent_of_root◁, child_right_not_a_rightmost_descendent◥, child_right_rightmost_descendent_of_non_root◣, child_right_rightmost_descendent_of_root▷), the other bits are used to store my_list_begin(). The bit2 of next_ stores the color of the node (reflected on the color of table cells' borders), the bit1 of next_ stores whether my_list_begin() is 0b1-tagged, the bit0 of next_ stores whether next() is 0b1-tagged, the other bits are used to store next().

Each element_t(int)'s corresponding accumulated_storage_t comes with the navigator portion (prev and next pointers) to be ready to form a circular doubly linked list. find_by_monotonic_predicate

element_t(std::string)s' .size()s are accumulated by std::size_t operator+(std::size_t, std::size_t). Use .find_by_monotonic_predicate([&](){ ... }) to find the first element where the accumulated_storage_t(std::size_t) (accumulated string size so far) starts to be greater than the given character index, i.e., the element that contains the the given character indexth character. Note that the hierarchical structure saves time by skiping substructures that don't contain the results.

find_by_heap_predicate

element_t(int)s are accumulated into their std::ranges::max. Use .find_by_heap_predicate(jts_output, [&](){ ... }) to eagerly find all elements greater than or equal to the given threshold.
